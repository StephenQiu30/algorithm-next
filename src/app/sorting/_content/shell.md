# 希尔排序 (Shell Sort)

希尔排序（Shell Sort）是插入排序的一种更高效的改进版本，也称为**递减增量排序算法**。它是计算机科学历史上第一批突破 $O(n^2)$ 屏障的排序算法之一。

## 1. 原理剖析
常规插入排序由于每次只能将元素移动相邻的一格，因此在遇到位于数列后端的极小值时，需要经过极多的比较和腾挪。希尔排序通过引入“增量（Gap）”的概念，将相互距离为 Gap 的元素编为一组进行插入排序，使得数据能实现较大步幅的跳跃。随着 Gap 逐渐缩小至 1，整个数列已近乎完全有序，最后一次的微调即可迅速完成。

## 2. 核心步骤
1. 选择一个递减增量序列 $t_1, t_2, \dots, t_k$，其中必须满足最后一个增量 $t_k = 1$。
2. 按照当前的增量步长 $Gap$，将数组逻辑上分割成 $Gap$ 个交错的独立子序列。
3. 对每一个子序列分别进行标准的插入排序。
4. 缩减 $Gap$ (通常直接折半)，重复上述过程，直到 $Gap=1$ 时的最后一趟整体插入排序完成。

## 3. 复杂度与稳定性
- **最佳时间复杂度**: 依赖于增量序列的选择，对于特定数列可达 $O(n \log n)$。
- **最坏时间复杂度**: 取决于增量序列（希尔原稿 $Gap = Gap/2$ 为 $O(n^2)$，Hibbard 序列为 $O(n^{1.5})$）。
- **平均时间复杂度**: 约为 $O(n^{1.3})$ 到 $O(n^{1.5})$。
- **空间复杂度**: $O(1)$。
- **稳定性**: **不稳定**。因为在分组跳跃排序时，相同数值但属于不同小组的元素，由于在不同小组内被分别排序，极易发生相对位置错乱。

## 4. 适用场景
- **中等规模数据**：在没有内置高阶排序可用的底层语言实现里，由于希尔排序只需要极短的代码（几乎不用额外函数或递归），它在嵌入式或系统编程底层备受青睐。
- **严格受限的内存与栈空间**：希尔排序既不需要 $O(n)$ 的辅助数组，也不需要类似快排的 $O(\log n)$ 递归栈空间。

## 5. 示例代码
以下是使用经典折半增量的希尔排序 TypeScript 实现：

```typescript
function shellSort(arr: number[]): number[] {
  const n = arr.length;
  // 初始增量设定为数组长度的一半，随后每次折半
  for (let gap = Math.floor(n / 2); gap > 0; gap = Math.floor(gap / 2)) {
    
    // 对每个组进行插入排序
    for (let i = gap; i < n; i++) {
      const temp = arr[i];
      let j = i;
      
      // 跨步长的大跳跃寻找并替换
      while (j >= gap && arr[j - gap] > temp) {
        arr[j] = arr[j - gap];
        j -= gap;
      }
      arr[j] = temp;
    }
  }
  return arr;
}
```
