# 选择排序 (Selection Sort)

选择排序（Selection Sort）是一种简单直观的比较排序算法。它的工作原理是：第一次从待排序的数据元素中选出最小（或最大）的一个元素，存放在序列的起始位置，然后再从剩余的未排序元素中寻找到最小（大）元素，然后放到已排序的序列的末尾。

## 1. 原理剖析

选择排序的核心思想非常直白：**总是选出剩余元素中的最值，并将其归位**。在这个过程中，数组实际上被分成了两个部分：前部分是已排序的极值元素，后部分是未排序的乱序元素。

## 2. 核心步骤

1. 初始化时，将前部视为已排序序列（初时为空），后部全部为待排序序列。
2. 遍历整个待排序序列，记录下最小（或最大）元素的位置。
3. 遍历结束后，将找到的最小元素与待排序序列的第一个元素交换位置，此时已排序序列长度增加一。
4. 重复步骤 2 和步骤 3，直到待排序序列中只剩下一个元素（此时它必然是最大/最小值，无需再排）。

## 3. 复杂度与稳定性

- **最佳时间复杂度**: $O(n^2)$。即使数组已经完全有序，选择排序依然会固执地遍历寻找最小值。
- **最坏时间复杂度**: $O(n^2)$。当数组逆序时同样是两层循环完全扫描。
- **平均时间复杂度**: $O(n^2)$。
- **空间复杂度**: $O(1)$。原地交换，仅需要记录最小值索引的空间。
- **稳定性**: **不稳定**。在这寻找最小值和交换的过程中，原本相等的元素相对位置可能会被破坏（例如：序列 `5 8 5 2 9`，第一遍找到极小值2，2与第一个5交换后，第一个5就跑到了第二个5后面）。

## 4. 适用场景

选择排序的数据移动操作（交换）最少，一共只会有 $O(n)$ 次交换。

- **对交换操作成本极高的数据**：如果排序对象的每个元素结构极其庞大，移动/拷贝成本远大于比较操作，选择排序的交换频率低于冒泡或插入。
- **硬件限制极强的嵌入式系统**：需要绝对稳定的常数级别额外空间，并希望对内存写操作最小化（例如向 Flash 存储写入块）。

## 5. 示例代码

以下是选择排序的 TypeScript 实现示例：

```typescript
function selectionSort(arr: number[]): number[] {
  const n = arr.length
  for (let i = 0; i < n - 1; i++) {
    // 默认未排序片段的第一个元素为当前最小值
    let minIndex = i

    // 在剩余元素中寻找真正的最小值索引
    for (let j = i + 1; j < n; j++) {
      if (arr[j] < arr[minIndex]) {
        minIndex = j
      }
    }

    // 如果真正的最小值不是初始元素，进行交换
    if (minIndex !== i) {
      const temp = arr[i]
      arr[i] = arr[minIndex]
      arr[minIndex] = temp
    }
  }
  return arr
}
```
