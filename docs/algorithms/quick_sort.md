# 算法讲解

## 介绍
快速排序（Quick Sort）是一种极其高效的排序算法，基于“分治”策略。它是实际应用中最常用的排序算法之一。

它的核心思想是：**“选个基准，然后左右站队”**。
1. **选基准 (pivot)**：从数组中选择一个元素作为基准。
2. **分区 (Partition)**：重新排列数组，将所有比基准小的元素移动到其左边，比基准大的元素移动到其右边。此时，基准元素就在其最终的正确位置上。
3. **递归 (Recursion)**：分别对基准左侧和右侧的两个子数组重复上述过程，直到每个子数组都排序完毕。

## 算法步骤
1. **选择基准**：从数列中挑出一个元素作为“基准”。
2. **分区操作**：遍历数组，将比基准小的元素交换到左边，大的交换到右边。
3. **递归排序**：递归地对基准左边的子序列和右边的子序列进行快速排序。
4. **结束条件**：子序列长度为 0 或 1 时停止递归。

### 示例流程
以数组 `[5, 3, 8, 4, 2]` 为例，选择第一个元素 `5` 为基准：
- **分区**：
  - 比较 3 和 5，3 < 5，保持在左边。
  - 比较 8 和 5，8 > 5，保持在右边。
  - 比较 4 和 5，4 < 5，交换到左边。
  - 比较 2 和 5，2 < 5，交换到左边。
  - 完成后数组可能变为 `[3, 4, 2, 5, 8]`（5 归位）。
- **递归**：对左侧 `[3, 4, 2]` 和右侧 `[8]` 分别继续进行上述操作。

## 复杂度分析
- **时间复杂度**：
  - **最好情况**：$O(n \log n)$。当每次选择的基准都能平分数组时。
  - **最坏情况**：$O(n^2)$。当数组已是有序或接近有序，且每次选择第一个或最后一个元素作为基准时（导致分区极度不均衡）。
  - **平均情况**：$O(n \log n)$。
- **空间复杂度**：$O(\log n)$。主要为递归调用栈的消耗。
- **稳定性**：**不稳定**。在分区交换过程中，相等元素的相对位置可能会发生改变。

## 优缺点
### 优点
- **速度极快**：在平均情况下，是所有内部排序算法中速度最快的一种。
- **原地排序**：不需要像归并排序那样占用大量额外内存。

### 缺点
- **最坏情况性能差**：若基准选择不当，时间复杂度会退化到 $O(n^2)$。
- **不稳定**：不适合需要保持相等元素原始相对顺序的场景。

## 深度洞察
### 1. 基准值（Pivot）的生与死
快速排序的性能高度依赖于基准值的选择。如果每次都选到最大或最小的元素（如在已排序数组中选第一个元素），复杂度会退化到 $O(n^2)$。这就是为什么现代实现通常采用**三数取中法（Median-of-three）**或**随机化基准**。

### 2. 为什么它通常比归并排序快？
虽然两者平均都是 $O(n \log n)$，但快排在实际运行中通常更快，原因有二：
- **原地排序（In-place）**：减少了内存分配和频繁的数据拷贝开销。
- **缓存极其友好**：分区过程具有非常好的顺序访问特性，能充分利用 CPU 的一级和二级缓存。

### 3. 混合动力：Introsort
许多编程语言（如 C++）的 `std::sort` 实际上是快排的加强版——**内省排序（Introsort）**。它开始时使用快排，如果发现递归深度过深（可能退化），则自动切换到堆排序，从而保证了最坏情况下的 $O(n \log n)$。

## 示例代码 (JavaScript)
```javascript
function quickSort(arr, left = 0, right = arr.length - 1) {
  if (left < right) {
    let pivotIndex = partition(arr, left, right);
    // 递归排序左子数组
    quickSort(arr, left, pivotIndex - 1);
    // 递归排序右子数组
    quickSort(arr, pivotIndex + 1, right);
  }
  return arr;
}

function partition(arr, left, right) {
  // 选择最右侧元素作为基准（也可以随机选择）
  let pivot = arr[right];
  let i = left;
  for (let j = left; j < right; j++) {
    if (arr[j] < pivot) {
      [arr[i], arr[j]] = [arr[j], arr[i]];
      i++;
    }
  }
  [arr[i], arr[right]] = [arr[right], arr[i]];
  return i;
}
```
