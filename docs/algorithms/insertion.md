# 插入排序 (Insertion Sort)

插入排序（Insertion Sort）的代码实现虽然没有冒泡排序和选择排序那么简单粗暴，但它的原理应该是最容易理解的，因为它的工作逻辑与**我们常人打扑克牌理牌的过程**一模一样。

## 1. 原理剖析
抓到一张新牌时，我们会习惯性地从右向左（或从左向右）看手中已经理好的牌，找到这张新牌应该插入的位置并将其推入。插入排序就是维护一个始终有序的前缀，然后将未排序的数据不断提取并向前“插入”的过程。

## 2. 核心步骤
1. 将待排序序列的第一个元素视为一个已排序的有序片段，把第二个元素到最后一个元素当成是未排序区。
2. 提取出未排序区的第一个元素，暂存它的值。
3. 从后向前扫描当前的有序片段，如果遇到了比暂存值大的元素，就将该元素向后挪动一位空出位置。
4. 直到遇到小于或等于暂存值的元素，或者到达了数组开头，将暂存值放入空出的位置。
5. 重复提取和插入的操作，直到所有未排序元素都被插入到了有序片段中。

## 3. 复杂度与稳定性
- **最佳时间复杂度**: $O(n)$。当数据已经基本有序时，新元素只需要与前一个元素比较一次即可确定位置。
- **最坏时间复杂度**: $O(n^2)$。当数据属于完全逆序时，每个新元素都要一直冒泡到数组最前面。
- **平均时间复杂度**: $O(n^2)$。
- **空间复杂度**: $O(1)$。
- **稳定性**: **稳定**。从后向前比较，遇到相等的元素我们停止比较将其放在后面，不改变原有的相对顺序。

## 4. 适用场景
插入排序实际上是非常重要且不可被忽视的排序算法，它常见于：
- **数据量极小的情况（通常 N < 15~50）**：在 V8 引擎（JavaScript）、Java 的 `Arrays.sort()` 中，当拆分的子数组足够小时，都会抛弃高级排序而降级改用插入排序，因为其常数项极小。
- **数据基本有序的情况**：插入排序在高度有序的数据面前能爆发出接近 $O(n)$ 级别的极高效率，常被作为混合排序算法（如 Timsort）的底层支撑。

## 5. 示例代码
以下是插入排序的 TypeScript 实现示例：

```typescript
function insertionSort(arr: number[]): number[] {
  const n = arr.length;
  // 从第二个元素开始，因为第一个元素默认是有序的
  for (let i = 1; i < n; i++) {
    // 取出当前需要对比的基准元素
    const current = arr[i];
    let j = i - 1;
    
    // 从已排序序列末尾向前扫描
    // 如果该元素大于取出元素，则向后挪动一位
    while (j >= 0 && arr[j] > current) {
      arr[j + 1] = arr[j];
      j--;
    }
    
    // 将取出元素插入到正确的位置
    arr[j + 1] = current;
  }
  return arr;
}
```
