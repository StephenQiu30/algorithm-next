# 算法讲解

## 介绍
堆排序（Heap Sort）是一种利用“堆”这种数据结构所设计的高效、原地排序算法。

**堆（Heap）** 是一种特殊的完全二叉树，具有以下特性：
- **最大堆**：每个节点的值都大于或等于其子节点的值，因此堆顶（根节点）始终是最大值。
- **最小堆**：每个节点的值都小于或等于其子节点的值，堆顶始终是最小值。

在堆排序中，我们通常使用数组来表示堆：
- 对于索引为 `i` 的节点：
  - 左子节点索引：`2 * i + 1`
  - 右子节点索引：`2 * i + 2`
  - 父节点索引：`(i - 1) / 2`

## 算法步骤
1. **构建最大堆**：将无序的待排序数组构建成一个最大堆。此时，数组的第一个元素就是最大值。
2. **交换与缩小堆**：将堆顶元素（最大值）与堆末尾的元素交换。此时最大值已归位到数组末尾。
3. **重新堆化**：将堆的大小减 1，并对新的堆顶元素进行“下沉”调整（Heapify），使其重新满足最大堆特性。
4. **重复循环**：不断重复交换和堆化过程，直到堆的大小缩小为 1，此时数组完全有序。

### 示例流程
以数组 `[4, 10, 3, 5, 1]` 为例：
- **第一步：构筑最大堆** -> `[10, 5, 3, 4, 1]`
- **第二步：交换堆顶 (10) 与末尾 (1)** -> `[1, 5, 3, 4, 10]`
- **第三步：调整堆** -> 重新将 `[1, 5, 3, 4]` 调整为最大堆 `[5, 4, 3, 1]`
- **第四步：继续交换** -> `5` 与 `1` 交换 -> `[1, 4, 3, 5, 10]`
- **最终**完成排序：`[1, 3, 4, 5, 10]`

## 复杂度分析
- **时间复杂度**：
  - **最好情况**：$O(n \log n)$。
  - **最坏情况**：$O(n \log n)$。
  - **平均情况**：$O(n \log n)$。
- **空间复杂度**：$O(1)$。原地排序，不需要额外空间。
- **稳定性**：**不稳定**。在交换堆顶和末尾元素的过程中，相等元素的相对位置可能会改变。

## 优缺点
### 优点
- **性能优秀且稳定**：时间复杂度无论如何都是 $O(n \log n)$。
- **空间效率极高**：原地排序，空间复杂度为 $O(1)$。
- **适合 Top K 问题**：可以非常快速地找到数据流中的最大或最小若干个元素。

### 缺点
- **由于需要频繁调整堆**：缓存局部性较差，实际运行速度通常慢于快速排序。
- **不稳定**：不适合需要保持稳定性的排序场景。

## 深度洞察
### 1. 绝对的稳定性（性能上）
与快速排序不同，堆排序在最坏情况下的时间复杂度依然是硬性的 $O(n \log n)$。它没有任何“病态”输入会导致其性能退化，这使得它在对**最差执行时间（WCET）**有严格要求的实时系统中非常受欢迎。

### 2. 优先队列（Priority Queue）
堆排序背后的核心结构——“堆”，是实现**优先队列**的标准方式。在许多算法（如 Dijkstra 最短路径算法、Huffman 编码）中，我们并不需要对整个数组排序，而只是不断地获取其中的最大或最小值，这时堆的优势就体现得淋漓尽致。

### 3. 为什么它不如快排流行？
尽管堆排序在理论上很均衡，但在现代计算机架构下，它的实际速度通常慢于快排。原因在于堆的操作需要在数组中“跳跃式”访问（父节点 `i` 到子节点 `2i+1`），这会导致**缓存命中率（Cache Hit Rate）**较低。而快排的分区操作主要是顺序访问，更加契合 CPU 的缓存预取机制。

## 示例代码 (JavaScript)
```javascript
function heapSort(arr) {
  const n = arr.length;

  // 1. 构建最大堆
  for (let i = Math.floor(n / 2) - 1; i >= 0; i--) {
    heapify(arr, n, i);
  }

  // 2. 交换并调整
  for (let i = n - 1; i > 0; i--) {
    [arr[0], arr[i]] = [arr[i], arr[0]]; // 堆顶最大值换到末尾
    heapify(arr, i, 0); // 调整剩余部分
  }
  return arr;
}

function heapify(arr, n, i) {
  let largest = i;
  let left = 2 * i + 1;
  let right = 2 * i + 2;

  if (left < n && arr[left] > arr[largest]) largest = left;
  if (right < n && arr[right] > arr[largest]) largest = right;

  if (largest !== i) {
    [arr[i], arr[largest]] = [arr[largest], arr[i]];
    heapify(arr, n, largest); // 递归调整受影响的子树
  }
}
```
