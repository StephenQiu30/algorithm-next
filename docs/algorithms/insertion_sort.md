# 算法讲解

## 介绍
插入排序（Insertion Sort）是一种简单直观的排序算法。它的工作方式类似于我们打牌时的整理牌序：**将待排序序列分为“已排序”和“未排序”两部分，不断地从未排序部分取出元素，将其插入到已排序部分的正确位置。** 

这就好比你左手拿着已经排好序的牌，右手不断从桌上抓起一张新牌，通过与左手中的牌从后往前比较，找到合适的位置插进去。

## 算法步骤
1. **初始状态**：将第一个元素视为已排序序列，其余元素视为未排序序列。
2. **取元素**：从未排序序列中取出第一个元素，称为“待插入元素”。
3. **比较与移动**：从已排序序列的末尾开始，依次与待插入元素比较。如果已排序序列中的元素大于待插入元素，则将该元素后移一位。
4. **插入**：重复步骤 3，直到找到小于或等于待插入元素的位置，将待插入元素插入该位置。
5. **循环**：重复上述步骤，直到所有未排序元素都处理完毕。

### 示例流程
以初始数组 `[5, 2, 4, 6, 1, 3]` 为例：
- **初始状态**：`[5]` 是已排序部分，`[2, 4, 6, 1, 3]` 是未排序部分。
- **第一轮**：待插入元素 `2`。`2 < 5`，`5` 右移，`2` 插入。数组变为 `[2, 5, 4, 6, 1, 3]`。
- **第二轮**：待插入元素 `4`。`4 < 5`，`5` 右移；`4 > 2`，此时找到位置插入。数组变为 `[2, 4, 5, 6, 1, 3]`。
- **第三轮**：待插入元素 `6`。`6 > 5`，直接放在末尾。数组变为 `[2, 4, 5, 6, 1, 3]`。
- **第四轮**：待插入元素 `1`。依次与 `6, 5, 4, 2` 比较并右移，插入首位。数组变为 `[1, 2, 4, 5, 6, 3]`。
- **第五轮**：待插入元素 `3`。找到 `2` 和 `4` 之间的位置插入。数组变为 `[1, 2, 3, 4, 5, 6]`。

## 复杂度分析
- **时间复杂度**：
  - **最好情况**：$O(n)$。当输入数组已经是正序时，只需进行 $n-1$ 次比较。
  - **最坏情况**：$O(n^2)$。当输入数组是反序时，需要 $n(n-1)/2$ 次比较和移动。
  - **平均情况**：$O(n^2)$。
- **空间复杂度**：$O(1)$。原地排序。
- **稳定性**：**稳定**。如果两个元素相等，由于是从后向前扫描且只有在严格大于时才移动，相对位置不会改变。

## 优缺点
### 优点
- **实现简单**：算法容易理解。
- **稳定性好**：是稳定的排序算法。
- **对近乎有序的数据非常高效**：在这种情况下，时间复杂度接近 $O(n)$。
- **原地排序**：不占用额外空间。

### 缺点
- **效率较低**：对于大规模乱序数据，性能较差。

## 深度洞察
### 1. 自适应性（Adaptability）
插入排序是高度**自适应**的。如果输入数组已经部分有序，它的工作量会大幅减少。在完全有序的情况下，它只需要 $n-1$ 次比较和 0 次交换，达到线性的 $O(n)$ 时间复杂度。

### 2. 混合排序的“基石”
虽然插入排序的大规模性能不如快排或归并，但由于其**常数因子极小**且逻辑简单，在现代工业级排序算法中扮演着重要角色：
- **Timsort**（Python, Java, Android 默认排序）：当子数组规模较小时（通常 $n < 64$），会切换到插入排序。
- **Introsort**（C++ `std::sort`）：同样在递归到较小规模时使用插入排序。

### 3. 在线算法（Online Algorithm）
插入排序是一个**在线**算法，这意味着它可以在接收到数据的同时进行排序。这使得它非常适合处理流式数据，或者在数据不断增加的场景下维护一个有序列表。

## 示例代码 (JavaScript)
```javascript
function insertionSort(arr) {
  const n = arr.length;
  for (let i = 1; i < n; i++) {
    // 待插入元素
    let current = arr[i];
    // 已排序部分的最后一个元素索引
    let j = i - 1;
    // 从后向前扫描已排序部分
    while (j >= 0 && arr[j] > current) {
      arr[j + 1] = arr[j]; // 元素后移
      j--;
    }
    // 插入待插入元素
    arr[j + 1] = current;
  }
  return arr;
}
```
