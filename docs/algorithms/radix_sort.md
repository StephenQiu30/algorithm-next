# 算法讲解

## 介绍
基数排序（Radix Sort）是一种非比较型的排序算法。它的核心思想是：**按照数位（例如个位、十位、百位）依次进行排序。** 

基数排序通常从最低有效位（LSD）开始，依次对每个位置的数字进行排序（通常使用计数排序作为子过程）。通过多轮分配和收集过程，最终实现整体有序。它特别适合处理大量、数据范围相对集中的整数。

## 算法步骤
1. **寻找最大值**：找出待排序数组中的最大元素，以确定数字的最大位数。
2. **逐位排序**：从最低位（个位）开始，对每一位上的数字进行稳定排序（通常使用计数排序）。
3. **分配与收集**：
   - **分配**：根据当前位的数值，将元素分配到 0-9 对应的桶中。
   - **收集**：按照桶的顺序将元素重新串起来，形成新的序列。
4. **进位处理**：对下一个更高位重复上述过程，直到处理完所有位数。

### 示例流程
以数组 `[170, 45, 75, 90, 802, 24, 2, 66]` 为例：
- **按个位排序**：
  - 结果：`[170, 90, 802, 2, 24, 45, 75, 66]`
- **按十位排序**：
  - 结果：`[802, 2, 24, 45, 66, 170, 75, 90]`
- **按百位排序**：
  - 结果：`[2, 24, 45, 66, 75, 90, 170, 802]`
- **排序完成**。

## 复杂度分析
- **时间复杂度**：$O(n \times k)$。其中 $n$ 是元素个数，$k$ 是数据的最大位数。
- **空间复杂度**：$O(n + k)$。需要额外的空间来存储桶和临时计数数组。
- **稳定性**：**稳定**。由于在每一位的排序中使用的是稳定排序（如计数排序），基数排序整体是稳定的。

## 优缺点
### 优点
- **速度极快**：在特定数据范围（如身份证号、手机号排序）下，性能远超比较排序（$O(n \log n)$）。
- **稳定性好**：是稳定的排序算法。
- **非比较排序**：不依赖于元素间的直接比较。

### 缺点
- **适用范围局限**：主要适用于整数或定长字符串。
- **空间开销大**：需要额外的内存来存储计数辅助数组。

## 深度洞察
### 1. 突破 $O(n \log n)$ 的限制
在计算机科学中，基于**比较**的排序算法（如快排、归并）其理论下限是 $O(n \log n)$。而基数排序通过利用数字的内部结构，采用了“分配”而非“比较”的策略，从而在特定条件下（如 $k < \log n$）实现了更优的时间复杂度。

### 2. 硬件友好的算法
基数排序的逻辑非常适合并行处理。在 FPGA 或专用网络交换机中，基数排序常被用来对数据包头部进行高速排序，因为它不涉及复杂的条件分支（Branching），性能非常稳定。

### 3. 时间与空间的极致权衡
基数排序虽然快，但它是以**空间换时间**。由于它需要大量的计数空间（或桶空间），在处理位宽很大（如 64 位整数）但元素数量较少时，它可能远不如快速排序高效。因此，它通常被用于处理诸如电话号码、身份证号等定长且范围明确的数据。

## 示例代码 (JavaScript)
```javascript
function radixSort(arr) {
  if (arr.length === 0) return arr;

  // 1. 寻找最大值以确定位数
  const max = Math.max(...arr);
  
  // 2. 从个位开始遍历（exp 为 1, 10, 100...）
  for (let exp = 1; Math.floor(max / exp) > 0; exp *= 10) {
    countingSortByDigit(arr, exp);
  }
  return arr;
}

function countingSortByDigit(arr, exp) {
  const n = arr.length;
  const output = new Array(n);
  const count = new Array(10).fill(0);

  // 统计每个位的出现次数 (0-9)
  for (let i = 0; i < n; i++) {
    const digit = Math.floor(arr[i] / exp) % 10;
    count[digit]++;
  }

  // 累加 count，确定元素位置
  for (let i = 1; i < 10; i++) {
    count[i] += count[i - 1];
  }

  // 从后向前构建输出数组（保证稳定性）
  for (let i = n - 1; i >= 0; i--) {
    const digit = Math.floor(arr[i] / exp) % 10;
    output[count[digit] - 1] = arr[i];
    count[digit]--;
  }

  // 复制回原数组
  for (let i = 0; i < n; i++) {
    arr[i] = output[i];
  }
}
```
